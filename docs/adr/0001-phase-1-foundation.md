# ADR 0001: Phase 1 기반 아키텍처 결정

- 상태: 승인됨
- 날짜: 2026-03-20

## 문맥

제품은 경제·금융·지정학 콘텐츠 제작을 위한 운영형 대시보드이며, 뉴스 탐색, 공식 통계 검증, 시장 데이터 보조 자료 관리, 대본 생성, 캐릭터 일관성 이미지, 영상 준비, 검수/버전 관리, 다운로드까지 하나의 파이프라인으로 연결해야 한다.  
또한 한국어 UI, 프로젝트 단위 영속성, 근거 추적 가능성, 제공자 교체 가능성이 핵심 요구사항이다.

## 결정

### 1. 저장소 구조는 모노레포로 간다
- 프런트엔드와 백엔드는 동일 저장소에서 관리한다.
- 설계 문서도 코드와 함께 버전 관리한다.

### 2. 백엔드는 계층형 + Port-Adapter 구조를 채택한다
- `domain / application / infrastructure / presentation` 4계층으로 분리한다.
- 도메인 계층은 외부 제공자와 DB를 모른다.
- 외부 제공자 연동은 모두 포트 뒤에 둔다.

### 3. 프로젝트 중심 데이터 모델을 채택한다
- 프로젝트가 모든 산출물과 설정의 루트 aggregate 역할을 한다.
- 이슈, 통계, 시장 데이터, 스냅샷, 근거, 대본, 장면, 이미지, 영상, 작업 로그, 버전 이력이 프로젝트에 귀속된다.

### 4. 사실성은 공식 통계 우선 원칙으로 설계한다
- 숫자 주장에는 공식 통계 근거를 우선 연결한다.
- 시장 포털 데이터는 보조 자료로 구분한다.
- 근거 매핑은 스크립트 생성 이전부터 구축한다.

### 5. 장기 작업은 큐로 분리한다
- 이슈 수집, 대본 생성, 이미지 생성, 영상 번들 준비 등은 비동기 처리한다.
- 작업 상태와 로그를 저장하고 재시도를 지원한다.

### 6. 저장소는 로컬과 S3 호환 구조를 함께 고려한다
- 개발은 로컬 저장소를 기본으로 한다.
- 운영은 S3 호환 스토리지로 교체 가능해야 한다.

### 7. 프런트엔드는 한국어 전용 편집형 대시보드로 구성한다
- 사이드바 네비게이션을 기본으로 한다.
- 각 단계에서 수동 지시를 넣을 수 있어야 한다.
- 검수와 버전 복원을 UI 구조에 포함한다.

## 결과

### 장점
- 제공자 교체 비용이 낮다.
- 도메인 규칙 테스트가 쉽다.
- 프로젝트 단위 복원과 감사 추적이 용이하다.
- 프런트와 백엔드의 계약 관리가 명확하다.

### 비용
- 초기 파일 수와 구조가 늘어난다.
- 단순 CRUD보다 설계 비용이 높다.
- 실제 제공자 연동 시 adapter 구현량이 많아진다.

## 보류 사항

- 인증/권한 모델
- Alembic 마이그레이션 체계
- 실시간 작업 상태 스트리밍 방식
- 실제 Claude/통계/시장 데이터 제공자 인증 정책
